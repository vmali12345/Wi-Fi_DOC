129#define FF_MAX_LFN 255
130/* The FF_USE_LFN switches the support for LFN (long file name).
131/
132/ 0: Disable LFN. FF_MAX_LFN has no effect.
133/ 1: Enable LFN with static working buffer on the BSS. Always NOT thread-safe.
134/ 2: Enable LFN with dynamic working buffer on the STACK.
135/ 3: Enable LFN with dynamic working buffer on the HEAP.
136/
137/ To enable the LFN, ffunicode.c needs to be added to the project. The LFN function
138/ requiers certain internal working buffer occupies (FF_MAX_LFN + 1) * 2 bytes and
139/ additional (FF_MAX_LFN + 44) / 15 * 32 bytes when exFAT is enabled.
140/ The FF_MAX_LFN defines size of the working buffer in UTF-16 code unit and it can
141/ be in range of 12 to 255. It is recommended to be set 255 to fully support LFN
142/ specification.
143/ When use stack for the working buffer, take care on stack overflow. When use heap
144/ memory for the working buffer, memory management functions, ff_memalloc() and
145/ ff_memfree() in ffsystem.c, need to be added to the project. */
